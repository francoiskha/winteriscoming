Définition du responsive web design (RWD)

Get your content ready to go anywhere because it’s going to go everywhere.

Brad Frost
  • Responsive Web Design : présentation adaptée aux multiples résolutions d'écran
  • à l’aide d’une grille fluide et/ou de media queries (CSS)…
  • … sans oublier toutes les spécificités de chaque support (JS mostly required)

Nouveaux sélecteurs

…et autres peu usités

  • Cibler un élément du DOM dont l’attribut est égal à la valeur indiquée 
element[attribut=valeur] {}
  • Cibler plus finement un attribut 
element[attribut^=valeur] {} // ^= commence par
element[attribut$=valeur] {} // $= se finit par
element[attribut*=valeur] {} // *= contient
  • Exemple concret 
div[class^=poster_] {} // les DIV dont la classe commence par poster_

Nouveaux sélecteurs

…et autres peu usités

  • Pseudo-sélecteurs plus abstraits : ( n )th 
    Sélectionne le X(ième) éléments B de l'élément A
A B:nth-child(X) {}
  • Un sélecteur permettant notamment des formules plus complexes
    • X peut être une formule comme 2n+1 où n peut être égal à 0, 1, 2, etc. 
      X = 2*0+1 (=1)
X = 2*1+1 (=3)
X = 2*2+1 (=5)
tr:nth-child(2n+1) {}
  • Ou plus simplement :-) 
tr:nth-child(odd)  {}  // pair
tr:nth-child(even) {}  // impair

Propriété Display

Travestir un élément

  • Largement antérieur à CSS3
  • Les éléments peuvent être de deux types :
    • Block : largeur de 100%, provoque le retour à la ligne, permet des marges (haut/bas)
    • Inline : largeur égale au contenu, les éléments se positionnent les uns après les autres
  • … faux !
    • list-item (CSS1) : éléments block en inline isolé par des marges extérieures
    • inline-block (CSS2) : pour des éléments inline, mix entre Block et Inline
    • etc. (grid & flex en CSS3)
